## 入门问题

### Q1: PolyVault 是什么？

**A**: PolyVault 是一个去中心化的密码管理解决方案，具有以下特点：

| 特性 | 说明 |
|------|------|
| **去中心化** | 无需云服务器，设备间直接通信 |
| **端对端加密** | 所有数据在传输和存储时都加密 |
| **多平台支持** | 支持 Android、iOS、Windows、macOS、Linux |
| **硬件安全** | 支持 K 宝（USB/NFC/蓝牙）硬件密钥 |
| **插件化架构** | 可扩展的插件系统 |

---

### Q2: PolyVault 与其他密码管理器有什么区别？

**A**: 主要区别对比：

| 功能 | PolyVault | 1Password | Bitwarden |
|------|-----------|-----------|-----------|
| 数据存储 | 本地 + P2P 同步 | 云端 | 云端/自托管 |
| 加密方式 | AES-256-GCM + zk_vault | AES-256 | AES-256 |
| 硬件密钥 | ✅ K 宝 | ✅ YubiKey | ✅ YubiKey |
| 去中心化 | ✅ 完全 P2P | ❌ | ❌ |
| 开源 | ✅ MIT | ❌ | ✅ GPL |
| 插件系统 | ✅ | ❌ | ❌ |


## 账户与安全

### Q5: 忘记主密码怎么办？

**A**: 主密码无法恢复，但可以通过恢复密钥重置：

```
步骤 1: 找到恢复密钥
   └─ 检查首次设置时保存的位置
   └─ 通常保存在安全的地方（如保险箱）

步骤 2: 使用恢复密钥
   └─ 在登录界面点击"使用恢复密钥"
   └─ 输入 24 个恢复词
   └─ 验证成功后可设置新主密码

步骤 3: 重新加密数据
   └─ 所有凭证会用新主密码重新加密
   └─ 建议立即同步到其他设备
```

⚠️ **重要提醒**:
- 恢复密钥是唯一恢复途径，请妥善保管
- 如果没有恢复密钥，数据将无法恢复
- 建议将恢复密钥保存在物理安全的地方

---

### Q6: 主密码有什么要求？

**A**: 主密码安全要求：

| 要求 | 说明 |
|------|------|
| 最小长度 | 12 字符 |
| 推荐长度 | 16+ 字符 |
| 复杂度 | 大小写字母 + 数字 + 特殊字符 |
| 唯一性 | 不要与其他服务密码相同 |
| 避免 | 个人信息、字典词汇、连续字符 |

**密码强度示例**:
```
❌ 弱: password123
❌ 弱: John1990
✅ 中: MyP@ss2024!
✅ 强: K#9xL$mN2pQr@7wZ
✅ 最强: 考虑使用密码短语: "horse-battery-staple-correct"
```

---

### Q7: 生物识别安全吗？

**A**: 生物识别安全性说明：

| 方面 | 说明 |
|------|------|
| 本地存储 | 生物数据不离开设备 |
| 加密保护 | 使用设备安全芯片存储 |
| 备用验证 | 始终保留密码验证作为备用 |
| 可撤销 | 可随时禁用生物识别 |

**安全建议**:
- ✅ 启用生物识别方便日常使用
- ✅ 对于高敏感操作仍需要主密码
- ⚠️ 注意：某些司法管辖区可能强制解锁

---

### Q8: 数据如何加密存储？

**A**: PolyVault 采用多层加密：

```
┌─────────────────────────────────────────┐
│              加密层次架构                │
├─────────────────────────────────────────┤
│  第 1 层: 主密码派生                     │
│  └─ PBKDF2 (100,000 次迭代)             │
│                                         │
│  第 2 层: 数据加密密钥                   │
│  └─ AES-256-GCM                        │
│  └─ 每个凭证独立密钥                    │
│                                         │
│  第 3 层: 安全存储                       │
│  └─ zk_vault 硬件级加密                 │
│  └─ Secure Enclave / TEE               │
│                                         │
│  第 4 层: 传输加密                       │
│  └─ TLS 1.3 + 双向认证                 │
└─────────────────────────────────────────┘
```

### Q12: 如何生成强密码？

**A**: 内置密码生成器功能：

```dart
// 使用示例
final generator = PasswordGenerator(
  length: 20,
  includeUppercase: true,
  includeLowercase: true,
  includeNumbers: true,
  includeSymbols: true,
  excludeSimilar: true,  // 排除易混淆字符 (0/O, 1/l)
);

final password = generator.generate();
// 输出: K#9xL$mN2pQr@7wZxT
```

**生成选项**:
- 长度: 8-64 字符
- 字符集: 自定义
- 排除相似字符
- 排除歧义字符
- 易读密码（密码短语）

## 权限管理

### Q13: 三级权限体系是什么？

**A**: PolyVault 的三级权限说明：

| 级别 | 名称 | 能力 | 持续时间 |
|------|------|------|----------|
| L1 | 普通 | 查看已存储凭证 | 永久 |
| L2 | 增强 | 编辑、新增凭证 | 30分钟（可配置） |
| L3 | 最高 | 删除、导出、设置变更 | 5分钟（可配置） |

**权限提升流程**:
```
L1 → L2: 生物识别
L2 → L3: 生物识别 + K 宝验证
```

---

### Q14: 为什么权限会自动降级？

**A**: 自动降级是安全机制：

| 原因 | 说明 |
|------|------|
| 超时 | 高权限有有效期限制 |
| 应用切换 | 切换到其他应用后自动降级 |
| 屏幕锁定 | 设备锁定后降级到 L1 |
| 手动降级 | 用户主动降级 |

**配置调整**:
```
设置 → 安全 → 权限管理

L2 有效期: [10-60 分钟]
L3 有效期: [1-30 分钟]
自动降级: [开/关]
```

---

### Q15: 如何禁用自动授权？

**A**: 自动授权配置方法：

```
1. 进入设置 → 安全 → 自动授权
2. 查看已配置的规则列表

4. 选择"禁用"或"删除"

或全局禁用:
```

## K 宝使用

### Q16: 支持哪些类型的 K 宝？

**A**: K 宝类型支持：

| 类型 | 连接方式 | 特点 |
|------|----------|------|
| USB K 宝 | USB-A/C | 即插即用，物理按钮确认 |
| NFC K 宝 | NFC | 手机轻触，移动便捷 |
| 蓝牙 K 宝 | BLE | 无线连接，支持远程 |

**兼容品牌**:
- zk_vault 官方 K 宝
- YubiKey 系列
- 其他 FIDO2 兼容设备

---

### Q17: K 宝无法连接怎么办？

**A**: 故障排除步骤：

**USB K 宝**:
```
1. 检查 USB 接口是否松动
2. 尝试其他 USB 接口
3. 检查设备管理器是否识别
4. 更新 USB 驱动
5. 重启电脑
```

**NFC K 宝**:
```
1. 确保手机 NFC 已开启
2. 移除手机壳
3. 调整 K 宝位置
4. 重启手机 NFC
```

**蓝牙 K 宝**:
```
1. 确保蓝牙已开启
2. 检查 K 宝电量
3. 忘记设备后重新配对
4. 重启 K 宝
```

---

### Q18: K 宝丢失了怎么办？

**A**: 丢失处理步骤：

```
紧急措施:
1. 立即在另一设备上登录
2. 进入设置 → 安全 → K 宝管理
3. 移除丢失的 K 宝
4. 添加新的 K 宝

预防措施:
- 备份 K 宝（建议准备 2 个）
- 记录 K 宝序列号
- 启用 K 宝丢失通知
```

## Agent 白名单

### Q22: 什么是 Agent？

**A**: Agent 是 PolyVault 的服务组件：

```
Agent 类型:
├─ OpenClaw Agent    (主控制端)
├─ Browser Agent     (浏览器扩展)
├─ IoT Agent         (物联网设备)
└─ Third-party Agent (第三方集成)

Agent 功能:
• 请求凭证
• 存储数据
• 设备发现
• 远程控制
```

---

### Q23: 如何管理 Agent 白名单？

**A**: 白名单管理步骤：

```
查看白名单:
设置 → 安全 → Agent 白名单

添加 Agent:
1. 点击"添加 Agent"
2. 扫描 Agent 二维码
3. 选择允许的服务
4. 验证身份确认

移除 Agent:
1. 在白名单列表找到 Agent
2. 点击"移除"
3. 确认移除

白名单上限: 50 个 Agent
```

---

### Q24: Agent 请求被拒绝怎么办？

**A**: 故障排除：

```
检查项:
1. Agent 是否在白名单中
2. Agent 证书是否过期
3. 请求的服务是否被允许
4. 当前权限级别是否足够

解决方法:
- 将 Agent 添加到白名单
- 更新 Agent 证书
- 调整 Agent 权限设置
- 提升当前权限级别
```

## eCAL 通信

### Q25: eCAL 是什么？

**A**: eCAL 是 PolyVault 的通信中间件：

| 特性 | 说明 |
|------|------|
| 全称 | Eclipse Consensus and Adaptive Learning |
| 来源 | Eclipse 基金会开源项目 |
| 模式 | 发布-订阅 + 客户端-服务器 |
| 传输 | 共享内存 / UDP / TCP |
| 序列化 | Protobuf |

**优势**:
- 高性能（微秒级延迟）
- 零配置设备发现
- 自动传输优化
- 跨平台支持

---

### Q26: eCAL 端口被占用怎么办？

**A**: 端口配置方法：

```yaml
# ecal.ini
[network]
# 默认端口
udp_port = 14000
tcp_port = 14001

# 修改为可用端口
udp_port = 15000
tcp_port = 15001
```

**修改后**:
1. 重启 eCAL 服务
2. 更新所有设备的配置
3. 重新配对设备
