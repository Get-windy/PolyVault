## 1️⃣ 认证安全测试

### TC-SEC-AUTH-001: 密码哈希强度
**测试目的**: 验证密码使用bcrypt正确哈希
**预期结果**: 密码存储为$2a$10$...格式，cost factor ≥ 10
**优先级**: P0

### TC-SEC-AUTH-002: JWT Token安全
**测试目的**: 验证JWT配置安全
**预期结果**: 
- 算法: HS256或RS256
- 有效期: 合理（访问令牌15分钟，刷新令牌7天）
- 密钥: 环境变量，最小256位
**优先级**: P0

### TC-SEC-AUTH-003: 生物认证安全
**测试目的**: 验证生物识别数据不离开设备
**预期结果**: 指纹/面部数据仅本地处理，不上传服务器
**优先级**: P0

### TC-SEC-AUTH-004: K宝设备认证
**测试目的**: 验证硬件K宝认证安全性
**预期结果**: 
- 私钥永不离开K宝
- 签名操作在K宝内完成
- 支持国密SM2/SM3算法
**优先级**: P0

## 2️⃣ 数据加密测试

### TC-SEC-CRYPTO-001: 本地数据加密
**测试目的**: 验证SQLite数据库加密
**预期结果**: 
- 使用SQLCipher AES-256-GCM
- 密钥派生: PBKDF2, 100,000+ iterations
- 加密密钥存储在Keychain/Keystore
**优先级**: P0

### TC-SEC-CRYPTO-002: 通信加密
**测试目的**: 验证所有通信使用TLS 1.3
**预期结果**: 
- eCAL通信使用TLS
- API调用使用HTTPS
- 证书 pinning 实现
**优先级**: P0

### TC-SEC-CRYPTO-003: 密钥管理
**测试目的**: 验证密钥安全存储
**预期结果**: 
- 主密钥: 设备安全硬件存储
- 派生密钥: 每次启动重新派生
- 密钥轮换: 支持定期轮换
**优先级**: P1

## 3️⃣ 访问控制测试

### TC-SEC-ACCESS-001: 基于角色的访问控制
**测试目的**: 验证RBAC正确实现
**预期结果**: 
- 用户角色: user, admin, auditor
- 权限粒度: 读/写/执行/管理
- 资源隔离: 用户只能访问自己的数据
**优先级**: P0

### TC-SEC-ACCESS-002: 多设备授权管理
**测试目的**: 验证设备授权机制
**预期结果**: 
- 新设备需授权
- 可撤销设备访问
- 设备指纹绑定
**优先级**: P1

### TC-SEC-ACCESS-003: 审计日志
**测试目的**: 验证敏感操作记录
**预期结果**: 
- 记录: 登录、授权、数据访问
- 不可篡改: 签名或只读存储
- 保留期: 90天以上
**优先级**: P1

## 4️⃣ 通信安全测试

### TC-SEC-COMM-001: eCAL通信安全
**测试目的**: 验证eCAL消息安全
**预期结果**: 
- 消息签名: 防止篡改
- 消息加密: 防止窃听
- 序列号: 防止重放攻击
**优先级**: P0

### TC-SEC-COMM-002: API安全
**测试目的**: 验证REST API安全
**预期结果**: 
- 认证: JWT或mTLS
- 速率限制: 100 req/min
- 输入验证: 所有参数验证
**优先级**: P0

## 5️⃣ 安全头与配置

### TC-SEC-HEADER-001: 安全头配置
**测试目的**: 验证HTTP安全头
**预期结果**: 
```
Strict-Transport-Security: max-age=31536000; includeSubDomains
Content-Security-Policy: default-src 'self'

X-Content-Type-Options: nosniff
X-XSS-Protection: 1; mode=block
Referrer-Policy: strict-origin-when-cross-origin
```

**优先级**: P0

- 调试模式: 关闭
- 日志级别: WARN+

**优先级**: P0
